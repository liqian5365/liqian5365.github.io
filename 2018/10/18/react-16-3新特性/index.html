react 16.3新特性

react 16.3新特性。

多多关注新的东西。
componentWillMount 即将废除 ，使用componentDidMount 作为替代
componentWillUpdate 即将废除 ，使用componentDidUpdate 作为替代
componentWillReceiveProps 即将废除 使用 static getDerivedStateFromProps 作为替代

getDerivedStateFromProps 静态方法，不能访问this关键字，静态即前面有static的修饰
更新state的话只需要返回新的状态，return出去就可以，不需要this.setState(),如果不需要更新状态，返回null就可以了。而且返回机制和setstate一样，只需要返回需要更新的状态就可以，其他的状态保留即可。
注意点：要在构造函数里面初始化状态。
由于componentWillMount/ UNSAFE_componentWillMount是在render之前调用，所以就算在这个方法中调用setState也不会触发重新渲染（re-render）
如果同时定义了 getDerivedStateFromProps 和 componentWillReceiveProps，只有 getDerivedStateFromProps 会被调用，同时 React 还会打印出警告信息。
还有一种情况就是，当状态发生变化的时候需要执行回调，这时候你就可以用 componentDidUpdate。

##StrictMode
Strict mode 是新加入的组件，旨在引导你遵循最佳实践。把需要进行约束的组件簇放在它的下面就完事了
react 生命周期分为三个阶段名分别是：

  1. 挂载阶段
  2. 更新阶段
  3. 卸载阶段

1.挂载阶段

也就是组件的初始化阶段，将我们的组件插入到DOM中，只会发生一次
这个阶段的生命周期函数

  • constructor
  • getDerivedStateFromProps
  • render
  • componentDidMount

2.更新阶段

当组件的props发生变化，或调用setState方法， forceUpdate方法
这个阶段的生命周期函数调用有

  • getDerivedStateFromProps
  • shouldComponentUpdate
  • render
  • getSnapshotBeforeUpdate
  • componentDidpdate

3.卸载阶段

componentWillUnMount() 组件卸载方法里面不要再写setState()方法，因为不会再出发render函数
在这里主要做一些定时器的清楚，内存的回收
