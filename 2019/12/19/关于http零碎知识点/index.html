关于http零碎知识点

Request Method:OPTIONS初窥CORS

浏览器在某些请求中会在正式通信前会增加一次HTTP查询请求，也叫预检请求。浏览器先咨询服务器，当前网页所在的域名是否在服务器许可名单之中，以及
可以使用哪些HTTP动词和头信息字段，只有得到肯定的答复，浏览器才会发出正式的XMLHttpRequest请求，否则就会报错

OPTIONS请求方法的主要用途有两个：

  1. 获取服务器支持额HTTP请求方法，也是黑客经常使用的方法。
  2. 用来检查服务器的性能，
  1. 是否禁止缓存
    是否禁止缓存，cache-control：no-store 和 Pragma: no-cache都可以禁止缓存
    cache-control：no-store是http 1.1提供的，Pragma可以兼容到http1.0和1.1
  2. 是否检查本地副本是否过期
    cache-control 和 must-revalidate来控制
    no-chache告诉浏览器、缓存服务器，不管本地副本是否过期，使用资源富本前，一定要到源服务器进行副本有效性校验
    must-revalidate：告诉浏览器，缓存服务器，本地副本过期前，可以使用本地副本，本地副本一旦过期，必须去源服务器进行有效校验。

  3. 本地副本是否过期
    首先了解缓存的过期机制：

    • max-age=,表示资源能够被缓存的最大时间，经常会和must-revalidate一起使用。

    • 如果不含max-age属性，则去查看是否包含expires属性，通过比较expires的值和头里面Date属性的值来判断是否缓存还有效。
